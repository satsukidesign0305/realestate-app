# CLAUDE.md

## プロジェクト概要

このプロジェクトは、不動産アプリ（realestate-app）です。不動産情報の管理、検索、表示を行うためのアプリケーションを開発しています。ユーザーが物件情報を効率的に扱えるようにすることを目指しています。

## 技術スタック

- **フロントエンド**: React (想定)
- **バックエンド**: Node.js (想定)
- **データベース**: MongoDB または PostgreSQL (想定)
- **その他**: Git, GitHub

プロジェクトが初期段階のため、技術スタックは開発中に決定されます。

## フォルダ構造

プロジェクトのフォルダ構造は以下の通りです（初期状態）：

```
realestate-app/
├── CLAUDE.md (このファイル)
├── src/ (ソースコード)
├── public/ (静的ファイル)
├── package.json (依存関係)
└── README.md (プロジェクト説明)
```

## 開発ルール

- コードは読みやすく、保守しやすいものにしてください。
- コメントを適切に記載し、複雑なロジックは説明を加えてください。
- テストコードを積極的に書いてください。

## Git運用ルール

- **コードを変更するたびに、必ずコミットして GitHub にプッシュしてください。** ローカルだけに留めず、変更は常にリモートリポジトリに反映することを徹底してください。
- コミットメッセージは明確で、変更内容を簡潔に記述してください（例: `feat: 物件検索フィルター機能を追加`）。
- ブランチは機能ごとに作成し、`main` ブランチにマージする前にレビューを行ってください。
- `main` ブランチへの直接プッシュは避け、プルリクエスト経由でマージしてください。
- プッシュ前に `git status` で変更内容を確認し、不要なファイル（`.env` 等）が含まれていないことを確かめてください。
